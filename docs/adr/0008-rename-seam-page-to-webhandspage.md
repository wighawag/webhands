# Rename the verb-level seam type `Page` to `WebHandsPage`

webhands' own transport-neutral seam type (the eight-verb interface defined in `packages/core/src/seam.ts`, ADR-0003) was named `Page`, which collided head-on with Playwright's own `Page` type. The collision forced three modules (the hand host and both Playwright transports) to import the real Playwright page as `type Page as PwPage` purely to dodge the clash, and it actively misled readers: `HandContribution.verbs: Partial<Page>` read as a partial of Playwright's huge `Page` rather than a subset of webhands' eight seam verbs. We therefore renamed the seam type to `WebHandsPage` (exported in place of `Page`), dropped the now-unnecessary `PwPage` aliases (Playwright's `Page` is now imported directly where the live Node-side page is held), and disambiguated the `CONTEXT.md` glossary. This is a NAME-ONLY, behaviour-preserving change (the verbs, wire shape, hand contract, and trust model are byte-for-byte unchanged; the existing `core` suite is the regression proof) but a BREAKING one for consumers, since the exported type name changed; it ships as a major bump for `@webhands/core`. Runtime value/function identifiers that merely contain "Page" (`composePage`, `composeBuiltInPage`, `assertCompletePage`, the exported `makeRpcPage`) were deliberately left untouched: only the seam TYPE was renamed.
