# Eval ShellAdapter's stdin write can EPIPE when the agent command exits without reading stdin (2026-06-29)

`evals/src/agent-under-test.ts` `ShellAdapter.launch` does `child.stdin.write(stdin); child.stdin.end()` with NO `error` handler on `child.stdin`. A command that exits before reading its stdin (e.g. a bare `printf` with no `cat`) closes the pipe, so the write rejects with an unhandled `EPIPE` (`{errno:-32,code:'EPIPE',syscall:'write'}`). Real pi/claude agents read stdin so this never bites in practice; it surfaced only when a deterministic test used a non-reading command. Spotted while adding the CDP-endpoint-plumbing self-test; worked around in the test (drain stdin via `cat >/dev/null`). A small hardening (swallow stdin EPIPE) would make the adapter robust to any agent command. Out of scope for `eval-baseline-shared-driving-surface-over-cdp`.
